The pipeline, stage by stage

Seven stages run between your prompt and the first character on screen. Each one below is simulated with the exact example from the diagram: "What is gravity?"

1
Structure · Text

Tokenizer

Your words become integer IDs. The model never sees letters — only these numbers.

gravity → grav26110 ity879
What is gravity? → [2601, 318, 26110, 879, 30]

LLMs can't count letters in a word — because they never see letters.

2
Core computation

Embeddings

Each token ID is looked up in an embedding matrix and becomes a 4096-dimensional vector. Language becomes geometry.

grav→one point in 4096-dim space
[0.021, −0.184, 0.907, … ] × 4096 values (d_model = 4096)

Core computation

Transformer blocks

Multi-head self-attention — softmax(QKᵀ/√dk)·V — followed by a feed-forward network, each wrapped in Add & LayerNorm. Repeated 96+ times in GPT-4-class models. Every pass, the representation deepens.

FlashAttention fuses attention ops to cut HBM reads — the layers are fast; memory traffic is the tax.

4
Caching · Memory

KV Cache

Keys and Values for every prior token are cached per layer, so nothing is recomputed at each step. Without it, inference is impossibly slow.

time →   t₁   t₂   t₃   t₄   t
K,V rows appended every step — never recomputed

The catch: the cache grows linearly with context length. This is the main memory bottleneck.

5
Activation · Sampling

Sampling

The LM head projects to logits over a 128K+ token vocabulary, softmax turns them into probabilities — then the sampler picks. How you sample changes everything.

Greedy

always pick argmax

Top-K

sample from top K tokens

Top-P (Nucleus)

cumulative probability cutoff

Temperature

T→0 deterministic · T→1 creative
6
Models · Components

Speculative decoding

A small draft model guesses 4–5 tokens ahead. The large target model verifies them all in a single forward pass — accepting the correct prefix, rejecting the rest.

draft proposes → target verifies in one pass:
Grav✓ ity✓ is✗ a✗ fundamental+
≈ 5 tokens for the cost of 1 large-model pass
7
Structure · Text

Detokenize & stream

Accepted token IDs map back to text and stream to your screen, one by one.

[2610, 879, 318, 256, 17042, …] →

Notes · Annotations

The two bottlenecks nobody separates clearly enough

PHASE 1 — PREFILL

Compute-bound

All input tokens are processed in parallel. The GPU's arithmetic units are the limit. Quantization (INT8/INT4) and FlashAttention help here.

FLOPs utilization ▲ · memory pressure —
PHASE 2 — DECODE

Memory-bound

One token at a time, each step re-reading weights and a KV cache that grows linearly with context. Memory bandwidth is the limit.

HBM bandwidth ▲ · compute idles —
Optimizing one does nothing for the other. This is why inference cost is still the hardest unsolved problem in AI infrastructure.
