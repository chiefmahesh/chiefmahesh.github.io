01

Define the problem before touching data.

"Human Behaviour" is too broad to train on. A veteran narrows it to a task taxonomy first — because the taxonomy decides the data, and the data decides everything else.

Task Taxonomy — what the model must do
A
Behaviour Recognition
Classify observed behaviour into a formal taxonomy: habit loops (cue→routine→reward), avoidance patterns, approach motivation, procrastination archetypes, decision heuristics.
B
Behaviour Prediction
Given history + context, predict likely next action with calibrated confidence. "80% likelihood of task deferral when sleep < 6h and task ambiguity is high."
C
Intervention Generation
Suggest evidence-based nudges grounded in COM-B, Fogg Behaviour Model, and implementation-intention research — never generic advice.
D
Causal Explanation
Explain why a pattern exists using established frameworks (operant conditioning, cognitive dissonance, self-determination theory) with citations.
Success criteria — decided on day one
MetricTarget v1
Behaviour classification F1 (27-class)≥ 0.82
Prediction calibration (ECE)≤ 0.08
Framework citation accuracy≥ 95%
Harmful advice rate (red-team set)0.0%
p95 inference latency (8B, A10G)< 1.8s
🎯 Veteran rule #1: If you can't write the eval before the model exists, you don't understand the problem yet. The eval set is the spec. We write 300 golden test cases in week one — before a single GPU spins up.
02

Data collection — the 80% that decides the outcome.

A behavioural LLM lives or dies on data diversity: academic grounding + real-world messiness + longitudinal signal. Target: ~2.1B tokens for continued pre-training, ~180K examples for supervised fine-tuning.

📚
Academic Corpus
Psychology & behavioural-science literature: open-access papers (PsyArXiv, PubMed Central OA subset), textbooks with cleared licences, APA-style case formulations.
~900M tokens
🗂️
Validated Instruments
Item banks + scoring logic for BFI-2, HEXACO, GAD-7, PHQ-9, BIS/BAS — with expert-written interpretive narratives per score band (licensed where required).
~40M tokens
💬
Conversational Behaviour Data
De-identified coaching transcripts, CBT-style dialogue datasets (e.g. licensed counselling corpora), annotated with technique labels (Socratic questioning, reframing, validation).
~350M tokens
📈
Longitudinal Diaries
Consented habit-tracking and experience-sampling datasets (ESM/EMA studies) — the only source that teaches temporal behaviour dynamics: streaks, relapse curves, context triggers.
~120M tokens
🧪
Synthetic Scenarios
Frontier-model-generated behavioural vignettes, verified by psychologists. Covers rare patterns (compulsive checking, delayed-return frustration) that real data under-represents.
~500M tokens
⚖️
Counter-Examples & Red-Team Set
Deliberately harmful advice, pseudo-science (NLP eye-accessing, learning styles), manipulation scripts — labelled NEGATIVE so the model learns what not to say.
~190M tokens
⚠️ The consent line is absolute. Behavioural data is health-adjacent. Every conversational and diary source needs documented consent for ML training, IRB clearance where applicable, and PII stripping before it enters the pipeline — not after. One contaminated source poisons the entire model legally.
03

Processing & annotation — where quality is manufactured.

Raw tokens are worthless. The pipeline below turns them into training-grade data with a three-tier annotation system.

Cleaning pipeline (in order)
1
De-identification
presidio + custom NER pass for names, locations, employers, medical identifiers. Behavioural text is re-identification-prone — we run two independent PII models and take the union.
2
Deduplication
MinHash LSH at document level (datasketch), then semantic dedup with embedding clustering at 0.92 cosine threshold. Psychology corpora are heavily duplicated across sources.
3
Quality filtering
Perplexity filter (KenLM), pseudo-science classifier (fine-tuned DeBERTa flags 14 known myth categories), citation-integrity check against CrossRef.
4
Taxonomy tagging
Every chunk tagged: behaviour class (27 labels), framework lineage (COM-B / SDT / operant / CBT), evidence tier (RCT / observational / expert opinion / anecdote).
Three-tier annotation system
TierWhoWhatVolume
T1LLM-assistedBulk taxonomy labels, format normalisation100%
T2Trained annotatorsSFT pair quality, rubric scoring 1–7~25%
T3Licensed psychologistsGolden set, safety review, DPO preference pairs~4%
💡 Inter-annotator agreement is your canary. We require Krippendorff's α ≥ 0.75 on the behaviour taxonomy before scaling annotation. If experts can't agree on labels, the model can't learn them — fix the taxonomy, not the model.
04

Training — three phases, strict order.

Base model: Llama 3.1 8B. Big enough for nuanced behavioural reasoning, small enough to iterate weekly and deploy affordably. The three-phase recipe below is the industry-standard path for domain LLMs.

Phase AContinued Pre-Training (CPT)

Domain corpus (2.1B tokens) at low LR to inject behavioural knowledge without catastrophic forgetting. lr=1e-5, cosine decay, 15% replay of general web data to preserve base capabilities. Full-parameter on 8×A100 for ~4 days, or LoRA r=64 if budget-bound.

↓
Phase BSupervised Fine-Tuning (SFT)

180K instruction pairs across the 4 task types. QLoRA r=16, alpha=32, 4-bit NF4 via Unsloth — 3 epochs, effective batch 128. Chat template locked to a system prompt that enforces framework citation and confidence reporting.

↓
Phase CPreference Alignment (DPO)

22K psychologist-ranked preference pairs. beta=0.1 via TRL. Optimises for: evidence-grounded > plausible-sounding, calibrated > confident, "it depends + here's what it depends on" > premature certainty. This phase is what makes it feel like an expert.

train_behaviour_llm.py 
# Phase B — SFT with QLoRA (Unsloth)
from unsloth import FastLanguageModel
from trl import SFTTrainer, DPOTrainer

model, tok = FastLanguageModel.from_pretrained(
    "meta-llama/Llama-3.1-8B-Instruct",
    max_seq_length=4096, load_in_4bit=True,
)
model = FastLanguageModel.get_peft_model(
    model, r=16, lora_alpha=32,
    target_modules=["q_proj","k_proj","v_proj",
                    "o_proj","gate_proj","up_proj","down_proj"],
    lora_dropout=0.05, use_gradient_checkpointing="unsloth",
)

trainer = SFTTrainer(
    model=model, train_dataset=sft_ds,      # 180K pairs
    args=dict(per_device_train_batch_size=8,
              gradient_accumulation_steps=16,
              num_train_epochs=3, learning_rate=2e-4,
              lr_scheduler_type="cosine", warmup_ratio=0.03),
)
trainer.train()

# Phase C — DPO on expert preference pairs
dpo = DPOTrainer(model=model, beta=0.1,
                 train_dataset=pref_ds)     # 22K ranked pairs
dpo.train()
🧠 Veteran rule #2 — skip RLHF-PPO for v1. Full PPO needs a reward model, more compute, and is unstable at 8B scale. DPO gets you 90% of the alignment benefit with 20% of the complexity. Revisit PPO/GRPO only when you have >100K preference pairs and a trained reward model from the feedback loop (Stage 7).

* vs base model — catastrophic-forgetting guard

The four gates
G1
Golden Set (automated)
300 psychologist-written cases with reference answers. Scored by rubric-guided LLM-judge (GPT-4-class, temperature 0) + exact-match on taxonomy labels. Runs in CI on every checkpoint.
G2
Safety Gate (adversarial)
1,200 red-team prompts: crisis disclosure, manipulation requests, diagnosis-seeking, pseudo-science bait. Required behaviour: refuse + redirect to professionals. Zero tolerance.
G3
Capability Retention
MMLU + GSM8K + IFEval vs base model. Domain training must not cost more than 3 points of general capability — otherwise the CPT replay mix is wrong.
G4
Human Expert Panel
3 licensed psychologists blind-review 100 sampled responses quarterly. Elo-scored against the previous version. A version that loses the Elo match doesn't ship, regardless of automated scores.
06

Deployment — serving, guarding, observing.

An 8B model quantised to AWQ 4-bit serves comfortably on a single A10G at <1.8s p95. The architecture below is production-standard.

Serving stack
LayerChoiceWhy
QuantisationAWQ 4-bit~2.5× throughput, <1% quality loss vs FP16
Inference enginevLLMPagedAttention, continuous batching, LoRA hot-swap
Adapter servingmulti-LoRAShip weekly adapter updates without full redeploys
RAG layerChromaDB + BM25Grounds citations in the actual literature corpus
GuardrailsLlama Guard 3 + rulesPre + post filters: crisis detection, PII echo, scope
ObservabilityLangfuse + PrometheusPer-request traces, token costs, quality sampling
Non-negotiable production behaviours
1
Crisis escalation path
A dedicated classifier watches every input. Self-harm or acute-distress signals bypass the LLM entirely and return trained crisis-resource responses. This path is tested weekly.
2
Confidence surfacing
Every prediction ships with its calibrated confidence. The UI renders anything below 0.65 as "tentative" — the model is trained to say "I'm not sure" and the product honours it.
3
Canary deployment
New adapters go to 5% traffic for 72h. Automatic rollback if thumbs-down rate rises >1.5σ above the 14-day baseline.
4
Scope honesty
Trained refusal: "I analyse behaviour patterns; I don't diagnose conditions." Repeated on every session start. A behavioural LLM that plays therapist is a lawsuit with a UI.
⚠️ Never let the model be the only safety layer. Guardrails fail. Prompts leak. The crisis classifier, the output filter, and the scope rules are independent systems — defence in depth is the only defence.
07

The feedback flywheel — improving every single day.

v1 is the starting line. The flywheel below is the actual product: a closed loop where every real interaction makes next week's model measurably better.

Continuous Improvement Loop
What flows through the loop
1
Signal collection (continuous)
Thumbs up/down, regenerations, session abandonment, correction messages ("no, I meant…"), and expert spot-audits. All logged to Langfuse with full traces, consent-gated.
2
Labelling (daily batch)
Negative-signal conversations enter the T1→T3 annotation queue. Psychologists write the correct response for the worst 50 cases per week — these become gold DPO pairs.
3
Hard-case mining (weekly)
Embedding-cluster all failures; identify systematic weak spots (e.g. "model confuses avoidance with laziness"). Generate targeted synthetic data for exactly those clusters.
4
Retraining (weekly LoRA, quarterly full)
Weekly: new SFT + DPO data merged into an adapter refresh (~4 GPU-hours). Quarterly: full CPT+SFT+DPO re-run folding in all accumulated data.
5
Gate → canary → full traffic
Same four gates as Stage 5, then 5% canary for 72h, then rollout. Any regression auto-rolls back. The loop closes and begins again.
♻️ Veteran rule #3 — the moat is the loop, not the model. Anyone can fine-tune Llama. Nobody can copy 40 weeks of expert-labelled failure cases mined from your real traffic. The flywheel compounds; a static model depreciates from day one.

The Principle

"A domain LLM is not a model. It is a pipeline with a model inside it — and the pipeline is what improves every day."
